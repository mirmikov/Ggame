# Prometheus Tournament

Браузерный турнир по информатике для 9–11 классов. Организатор показывает общую арену на большом экране, а команды подключаются по коду со своих устройств и решают теоретические и кодовые задания.

## Режимы

### Отборочный тур — захват зоны

- организатор выбирает от 2 до 8 команд;
- организатор выбирает конкретный класс турнира и количество теоретических вопросов;
- в каждой команде находится несколько отдельных участников;
- игрок после входа выбирает команду и решает задания со своего устройства;
- ответы всех участников складываются в общий прогресс команды;
- команды стартуют по кругу вокруг центральной зоны;
- сложные теоретические и кодовые задания дают больше продвижения;
- команда, вошедшая в занятую зону, вытесняет прежнего владельца назад;
- удержавшая зону команда проходит в финал;
- в финал проходит примерно половина активных команд с округлением вверх;
- при завершении времени недостающие места определяются по прогрессу захвата, расстоянию до зоны и командным очкам.

### Финал — команда против команды

- участники выбирают сторону NexGen или OmniSoft;
- теория и код усиливают бойцов;
- две стороны сражаются на общей арене до окончания времени или уничтожения башни.

## Экраны

- **Организатор:** код лобби, составы команд, центральная зона, перемещения, захват, прошедшие команды и события.
- **Участник:** выбор команды, личные задания, редактор кода, общий командный прогресс и личная статистика.
- **Финал:** организатор видит арену, участники видят только задания и характеристики.

## Запуск без Docker

Требования: Go 1.22+, Node.js 20+.

## Деплой на домен

Production-сборка запускается через Docker Compose. В compose включён Caddy: он принимает внешний трафик на 80/443 и проксирует приложение внутрь контейнера на 8080. Поэтому фронтенд, API и WebSocket доступны с одного домена без указания порта.
Данные аккаунтов, серверов турниров и событий хранятся в PostgreSQL-контейнере с постоянным volume.

Подробная инструкция для чистого Ubuntu-сервера: [docs/FIRST_DEPLOY.md](docs/FIRST_DEPLOY.md).

1. Направьте A/AAAA-запись домена на сервер.
2. На сервере откройте порты 80 и 443.
3. Создайте `.env` рядом с `compose.yaml`:

```bash
SITE_ADDRESS=game.example.com
HTTP_PORT=80
HTTPS_PORT=443
POSTGRES_DB=ggame
POSTGRES_USER=ggame
POSTGRES_PASSWORD=replace-with-a-long-password
```

4. Запустите:

```bash
bash scripts/deploy.sh
```

После запуска приложение будет доступно по адресу `https://game.example.com`. Для локальной проверки без домена можно оставить `SITE_ADDRESS=:80` и открыть `http://localhost`.

### Backend

Откройте первый терминал:

```powershell
cd C:\Projects\Ggame-main\backend
$env:PORT="53124"
go run ./cmd/server
```

Число `53124` можно заменить на любой свободный порт. Терминал должен остаться открытым и не должен показывать `exit status 1`.

Проверка:

```text
http://localhost:53124/api/health
```

Ожидаемый ответ:

```json
{"status":"ok"}
```

### Frontend

Откройте второй терминал:

```powershell
cd C:\Projects\Ggame-main\frontend
Set-Content -Path .env.local -Value 'VITE_API_URL=http://localhost:53124'
npm.cmd install
npm.cmd run dev
```

Откройте адрес Vite, обычно:

```text
http://localhost:5173
```

Backend и frontend должны работать одновременно в двух терминалах.

### Подключение телефонов в одной Wi-Fi сети

Узнайте IPv4-адрес компьютера командой `ipconfig`, например `192.168.1.45`.

Backend:

```powershell
$env:PORT="53124"
go run ./cmd/server
```

Frontend:

```powershell
Set-Content -Path .env.local -Value 'VITE_API_URL=http://192.168.1.45:53124'
npm.cmd run dev -- --host
```

Команды открывают:

```text
http://192.168.1.45:5173
```

Все устройства должны находиться в одной сети. При необходимости разрешите Go и Node.js в брандмауэре Windows.

## Как провести отбор

1. Организатор создаёт лобби в режиме «Захват центральной зоны».
2. Организатор выбирает класс, число теоретических вопросов, число команд и число мест в каждой команде.
3. Каждый участник вводит своё имя, подключается по коду и выбирает команду.
4. Пустые команды при запуске будут пропущены.
5. После запуска участники решают задания, а ответы двигают общий маркер команды.
6. Тур автоматически завершается, когда в финал проходит заданное число команд.

## Ограничения MVP

- аккаунты, сессии, серверы турниров, состояние комнат и события турниров сохраняются в PostgreSQL;
- кодовые ответы проверяются сравнением с безопасными вариантами, а не запуском произвольного кода;
- в отборе участники подключаются отдельно, но прогресс команды общий;
- пустые команды в отборе пропускаются при запуске.
